# Model Card: Titanic Survival Prediction (AutoML)

## Обзор модели

| Параметр | Значение |
|----------|----------|
| **Название** | Titanic Survival Prediction |
| **Тип задачи** | Бинарная классификация |

| **Версионирование** | MLflow Model Registry |
| **Датасет** | Titanic (Kaggle) |
| **Целевая переменная** | `Survived` (0 — погиб, 1 — выжил) |

## Описание

Модель предсказывает вероятность выживания пассажира Титаника на основе социально-демографических признаков. Построена с помощью PyCaret (AutoML) — система автоматически перебирает 15+ алгоритмов (Logistic Regression, Random Forest, XGBoost, LightGBM, CatBoost и др.) и выбирает лучшую модель по метрике Accuracy.

Проект является учебным и демонстрирует полный production ML-пайплайн: мониторинг данных, автоматическое переобучение, версионирование моделей и A/B тестирование.

## Входные данные (Features)

| Признак | Тип | Описание |
|---------|-----|----------|
| `Pclass` | int | Класс каюты (1, 2, 3) |
| `Sex` | str | Пол (male/female) |
| `Age` | float | Возраст |
| `SibSp` | int | Кол-во братьев/сестёр или супругов на борту |
| `Parch` | int | Кол-во родителей/детей на борту |
| `Fare` | float | Стоимость билета |
| `Embarked` | str | Порт посадки (C/Q/S) |

## Выходные данные

| Поле | Тип | Описание |
|------|-----|----------|
| `prediction` | int | Предсказание: 0 (погиб) или 1 (выжил) |
| `probability` | float | Вероятность выживания (0.0–1.0) |

## Метрики качества

Метрики получены на тестовой выборке (20% от датасета, стратифицированное разбиение):

| Метрика | Значение | Описание |
|---------|----------|----------|
| **Accuracy** | ~0.82 | Доля правильных предсказаний |
| **Precision** | ~0.79 | Точность предсказания класса "выжил" |
| **Recall** | ~0.72 | Полнота предсказания класса "выжил" |
| **F1-Score** | ~0.75 | Гармоническое среднее Precision и Recall |
| **ROC-AUC** | ~0.86 | Площадь под ROC-кривой |

*Точные значения зависят от конкретного запуска AutoML и логируются в MLflow.*

## Обучение

### Процесс
1. **Загрузка данных** — Titanic dataset (891 запись для обучения)
2. **Предобработка** — PyCaret автоматически выполняет:
   - Imputation пропущенных значений (mean для числовых, mode для категориальных)
   - One-hot encoding категориальных признаков
   - Нормализация (опционально)
3. **Обучение** — `compare_models()` перебирает алгоритмы с 10-fold cross-validation
4. **Выбор лучшей модели** — по метрике Accuracy
5. **Сохранение** — `save_model()` в формате pickle + регистрация в MLflow

### Гиперпараметры
Подбираются автоматически PyCaret. Все параметры логируются в MLflow:
- `session_id`: фиксированный для воспроизводимости
- `log_experiment=True`: автоматический трекинг в MLflow
- `fold=10`: количество фолдов кросс-валидации

### Инфраструктура обучения
- **AutoML**: PyCaret 3.x
- **Трекинг**: MLflow Tracking Server
- **Оркестрация**: Apache Airflow DAG
- **Триггер**: обнаружение data drift (PSI > 0.2)

## Мониторинг и переобучение

### Data Drift Detection
- **PSI** (Population Stability Index): порог > 0.2
- **KS-тест** (Колмогоров-Смирнов): p-value < 0.05
- **Сдвиг среднего**: |mean_cur - mean_ref| > 1.0 * std_ref
- Проверка запускается ежедневно через Airflow DAG

### Переобучение
При обнаружении drift автоматически:
1. Запускается PyCaret AutoML на новых данных
2. Новая модель регистрируется в MLflow как **Staging**
3. Запускается A/B тест (20% трафика на новую модель)
4. При статистически значимом улучшении (p < 0.05) → переход в **Production**

## Ограничения и риски

### Данные
- Датасет исторический (1912 год) — модель не применима к современным предсказаниям
- Малый объём данных (891 запись) — ограниченная генерализация
- Пропущенные значения в `Age` (~20%) и `Cabin` (~77%, исключён из признаков)

### Модель
- Учебный проект — не предназначена для production-использования в реальных системах
- Переобучение на синтетическом drift (генерируется для демонстрации пайплайна)
- Метрики на тестовой выборке — не гарантируют performance на новых данных

### Этические аспекты
- Модель использует признак `Sex` — потенциальный bias (исторический: "женщины и дети первыми")
- При применении в реальных системах необходим аудит fairness-метрик
